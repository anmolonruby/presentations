CartoCSS is written in Javascript and runs primarily under node.js.

It can and has run in browsers instead, but there are no good renderers written for browsers yet and I'm not going to get into it.

CartoCSS tokenizes source, turns it into an AST, 'flattens' that AST, doing the work of inheritance, descendent selectors, expression evaluation, and so on.

The end result of this hard work is a flattened AST. Which is converted into the XML format Mapnik, the only supported renderer, uses.

We used the less.js parser because it had great error-catching support and a fast parser. It's likely the best JavaScript CSS parser around, though css-parser by TJ Holowaychuk is simpler and cooler.

CartoCSS has been partially ported to C++ in the carto-parser project. It's a long-term to get CartoCSS support into Mapnik itself.

We've had many discussions about standardizing the 'CartoCSS language' and probably the best product is at github.com/nvkelso/carto-css-map-styling

CartoCSS is different from GeoServer CSS because it's not related to SLD and tries very hard to support everything in Mapnik.

It's also written in Javascript and has a bunch of neat features.

CartoCSS is different from MapCSS because it tries very hard to support everything in Mapnik, and does not try to do multiple platforms.

But it does aim to support all datasources, not just OSM.
